# Translation maintenance notes

This is an operator-facing reminder for refreshing the Qt translation
catalogs. It supplements the existing `doc/translation_process.md`.

## When to run lupdate

Run `lupdate` whenever Qt source strings change in a way that
matters for translators — i.e. anywhere a `tr("…")` call's string
text was added, edited, or removed.

## How to run lupdate

`lupdate` ships with the Qt5 development tools.

### Linux (Debian/Ubuntu)

```
sudo apt-get install qttools5-dev-tools
cd /path/to/ratatoskr
lupdate src/qt -ts src/qt/locale/dash_en.ts
```

(Optional: also re-run for the other languages, but typically only
the source-language `dash_en.ts` is regenerated by lupdate; the
other `dash_*.ts` files get translation-merged later.)

### macOS

```
brew install qt@5
$(brew --prefix qt@5)/bin/lupdate src/qt -ts src/qt/locale/dash_en.ts
```

### Windows

Install Qt5 from the Qt installer, then run `lupdate.exe` from the
Qt installation's `bin/` against the source tree.

## What lupdate does

For each `tr("…")` string in `src/qt/`:

- Adds a `<source>` block to the `.ts` catalog
- Marks new strings as `<translation type="unfinished">` so
  translators see they need attention
- Marks strings that no longer appear in source as `obsolete`

The result is a `dash_en.ts` file that reflects the current source
of truth. Push that to GitHub and Transifex can pick it up.

## Note: catalog filename

The catalog files are still named `dash_*.ts` for historical
reasons (the fork lineage) — the `dash_locale.qrc` resource bundle
references them by exact filename, and the build system has these
filenames baked in. Renaming them to `ratatoskr_*.ts` is a
multi-file change touching the qrc, the Makefile.qt.include build
rules, and the lupdate output target. Deferred as a cosmetic
follow-up; the user-visible strings inside the .ts files are what
translators actually see.

## Stale `.xlf` file

`src/qt/locale/dash_en.xlf` is an older XLIFF-format export. The
build system uses the `.ts` files; the `.xlf` is for whoever
publishes to Transifex. It should be regenerated alongside `.ts`
when running lupdate, or just deleted if Transifex isn't in use.

## Skip lupdate? Acceptable for alpha

If you ship the alpha without re-running lupdate, English speakers
see the new strings fine (they're literal English in the source).
Non-English speakers see the new strings in English fallback while
the rest of the wallet stays translated. That's a reasonable
degraded experience for an alpha.

For v1.0 final, run lupdate so translators have something to work
with.
